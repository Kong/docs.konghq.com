Terminology

  • plugin: a plugin executing actions inside Kong before or after a request has been proxied to the upstream API.
    • {% if page.params.api_id %}
  • api: your upstream service placed behind Kong, for which Kong proxies requests to.
    • {% endif %} {% if page.params.service_id or page.params.route_id %}
  • service: The kong entity representing an external upstream API or microservice.
    • {% endif %} {% if page.params.route_id %}
  • route: The kong entity representing a way to map downstream requests to upstream services.
    • {% endif %} {% if page.params.consumer_id %}
  • consumer: a developer or machine using the api. When using Kong, a Consumer only communicates with Kong which proxies every call to the said, upstream api.
  • credential: a unique string associated with a consumer, also referred to as an API key.
    • {% endif %}
